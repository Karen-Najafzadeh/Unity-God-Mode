# فضاهای برداری و سینماتیک خطی

در **جلد اول، فصل ۵: فضاهای برداری و سینماتیک خطی**، ما از «دستور زبان» نحو C# به قوانین بنیادی واقعیت فضایی گذر می‌کنیم. این فصل برای بازسازی شهود مهندسی شما طراحی شده است تا به شما بیاموزد که چگونه یک کامپیوتر فضای سه‌بعدی را درک و دستکاری می‌کند و شما را از صرفاً «نوشتن اسکریپت» به عنوان معمار یک شبیه‌سازی فیزیکی ارتقا می‌دهد.

### [۱. اسکالرها در برابر بردارهای مختصاتی: دی‌ان‌ای فضا](/Volume-I-Mathematical-Foundations/Chapter-5-Vector-Spaces-and-Linear-Kinematics/5-1-Scalars-and-Coordinate-Vectors.md)
برای درک بردارها، ابتدا باید خویشاوند ساده‌تر آن‌ها را درک کنیم: **اسکالر**.

* **دانش علوم کامپیوتر:** در مغز ریزپردازنده، یک **اسکالر** یک نوع مقدار اولیه (مانند `int` یا `float`) است که مستقیماً در ثبات CPU ذخیره می‌شود. با این حال، یک **بردار** یک «آرایه بسته‌بندی‌شده» است—مجموعه‌ای از سه عدد اعشاری $(X, Y, Z)$ که با هم ذخیره می‌شوند تا سخت‌افزار بتواند عملیات «SIMD» (دستور واحد، داده‌های چندگانه) را روی آن‌ها انجام دهد.
* **مشکل اصلی:** کامپیوتر به طور طبیعی «شمال» یا «جلو» را درک نمی‌کند. اگر به کامپیوتر بگویید یک شخصیت در «۵» است، نمی‌داند که این یک مکان روی نقشه است یا سرعتی که شخصیت با آن می‌دود.
* **چگونه مشکل را حل می‌کند:** ما از **بردارهای مختصاتی** برای جدا کردن «اسم‌ها» از «فعل‌ها» در فضا استفاده می‌کنیم.
    * یک **بردار موقعیت** نشان‌دهنده یک نقطه خاص نسبت به مرکز جهان $(0,0,0)$ است.
    * یک **بردار جابجایی** نشان‌دهنده یک «جهت خالص» است—به شما می‌گوید چقدر حرکت کنید، نه اینکه در حال حاضر کجا هستید.

### [۲. مکانیک بردارها: ریاضیات حرکت](/Volume-I-Mathematical-Foundations/Chapter-5-Vector-Spaces-and-Linear-Kinematics/5-2-Vector-Mechanics-Algebraic-Operations.md)
هنگامی که این بردارها را داریم، باید بدانیم چگونه آن‌ها را با استفاده از عملیات جبری به هم «بچسبانیم».

* **جمع برداری:** وقتی یک بردار جابجایی را با یک بردار موقعیت جمع می‌کنید، مقصد جدیدی را محاسبه می‌کنید.
* **تفریق برداری (دید هوش مصنوعی):** اگر موقعیت نگهبان را از موقعیت بازیکن کم کنید، یک «بردار هدایت» به دست می‌آورید که مستقیماً از نگهبان به سمت بازیکن اشاره می‌کند.

**مثال: تعقیب هوش مصنوعی**
```csharp
// فرمول ریاضی برای "برای پیدا کردن بازیکن به کجا نگاه کنم؟"
// هدف - خود = جهت
Vector3 vectorToPlayer = playerTransform.position - transform.position;
```

### [۳. معیار فاصله اقلیدسی: اجتناب از «مالیات ریاضی»](/Volume-I-Mathematical-Foundations/Chapter-5-Vector-Spaces-and-Linear-Kinematics/5-3-Euclidean-Distance-Metrics.md)
محاسبه فاصله دشمن ممکن است ساده به نظر برسد، اما «مالیات» پنهانی بر عملکرد کامپیوتر شما دارد.

* **مشکل اصلی:** فرمول استاندارد **فاصله اقلیدسی** نیازمند محاسبه **ریشه دوم** است. در علوم کامپیوتر، ریشه‌های دوم گران هستند زیرا CPU باید یک فرآیند تکراری پیچیده را برای یافتن پاسخ اجرا کند.
* **راه‌حل "God Mode":** به جای استفاده از `Vector3.magnitude`، از `Vector3.sqrMagnitude` استفاده می‌کنیم. این مقدار فاصله *مجذورشده* را برمی‌گرداند. اگر فقط باید بدانید آیا دشمن «به اندازه کافی نزدیک» است، مقایسه فواصل مجذورشده یک عملیات سخت‌افزاری $O(1)$ است که بسیار سریع است.

**مقایسه کد:**
```csharp
// کند: یک محاسبه ریشه دوم انجام می‌دهد
if (Vector3.Distance(transform.position, player.position) < 5f) { ... }

// GOD MODE: تخصیص صفر، مقایسه با سرعت بالا
float sqrDist = (player.position - transform.position).sqrMagnitude;
if (sqrDist < 25f) { // 25، ۵ به توان دو است. برای CPU سریع‌تر است!
    Debug.Log("بازیکن در فاصله ۵ متری است.");
}
```

### [۴. نرمال‌سازی برداری: پروتکل «جهت خالص»](/Volume-I-Mathematical-Foundations/Chapter-5-Vector-Spaces-and-Linear-Kinematics/5-4-Vector-Normalization-Mechanics.md)
در فیزیک بازی، اغلب می‌خواهید بدانید به کدام سمت نگاه کنید بدون اینکه برایتان مهم باشد هدف چقدر دور است.

* **مفهوم:** **نرمال‌سازی** هر بردار را به یک **بردار واحد** تبدیل می‌کند—برداری با طول دقیقاً ۱.۰.
* **مشکل اصلی:** اگر بازیکن به صورت مورب حرکت کند (بالا و راست همزمان)، ریاضیات خام باعث می‌شود تقریباً ۱.۴۱ برابر سریع‌تر از حرکت مستقیم «بالا» حرکت کند. این یک باگ کلاسیک در بازی‌های آماتور است.
* **راه‌حل:** با نرمال‌سازی بردار حرکت، اطمینان حاصل می‌کنیم که بازیکن همیشه با سرعت دقیق ۱.۰ واحد در ثانیه حرکت می‌کند، صرف نظر از جهتی که رو به آن است.

### [۵. فضاهای مختصاتی و تبدیل‌های ساختاری](/Volume-I-Mathematical-Foundations/Chapter-5-Vector-Spaces-and-Linear-Kinematics/5-5-Coordinate-Spaces-Structural-Transformations.md)
در نهایت، این فصل به **توپولوژی‌ها** می‌پردازد—اینکه چگونه موتور واقعیت را سازماندهی می‌کند.

* **دانش:** Unity از یک **سیستم مختصات چپ‌دست** استفاده می‌کند. اگر دست چپ خود را باز کنید، شست شما $X$ (راست)، انگشت اشاره $Y$ (بالا) و انگشت میانی $Z$ (جلو) است.
* **سلسله‌مراتب چارچوب‌ها:** شما تفاوت بین **فضای محلی** (جایی که دست شما نسبت به بدنتان است) و **فضای جهانی** (جایی که دست شما نسبت به خورشید است) را یاد خواهید گرفت. تسلط بر این «تبدیل‌ها» به شما امکان می‌دهد اشیاء پیچیده‌ای مانند ماشین‌ها بسازید که در حین چرخش، چرخ‌هایشان متصل باقی می‌ماند.
